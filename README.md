Todo app monorepo with a Rust GraphQL backend and a Shadcn-based frontend.

## Architecture

This project follows **ports and adapters (hexagonal)**:

- **Ports:** `packages/jwt_port`, `packages/refresh_token_port`
- **Adapters:** `packages/jwt_jsonwebtoken`, `packages/redis`
- **Web/API:** `packages/web` (Axum + `async-graphql`)
- **Error handling:** `rootcause`

Frontend is based on a **Shadcn Admin template** (`packages/frontend`) and uses **KISS (`kiss-for-react`) state management** with an Elm-style flow (state + actions/reducers), especially in the todos feature.

## Database approach

- Schema source: `packages/db/atlas/schema.sql` (**Atlas**)
- Runtime migrations: `packages/db/migrations` (**Refinery**, embedded at startup)
- Type-safe query crate: `packages/db_queries` (**generated by Clorinde**)
- Custom DAL workflow: `make dal NAME=<migration_name>`
  - runs `atlas migrate diff`
  - copies newest SQL into `packages/db/migrations/V{N}__<name>.sql`
  - refreshes `packages/db/migrations/atlas.sum`

## Local development (HAProxy + /etc/hosts)

This repo uses a local dev proxy with custom hosts.

1. One-time (optional) to allow HAProxy on low ports without sudo each run:
   ```bash
   make proxy-setup-no-sudo
   ```
2. Sync host entries into `/etc/hosts`:
   ```bash
   ./scripts/dev-proxy/hosts.sh sync
   ```
   This manages:
   - `todoapp.localhost`
   - `api.todoapp.localhost`
3. Start full dev stack (proxy, backend, frontend, postgres, redis):
   ```bash
   make start
   ```

Default URLs:

- Frontend: `http://todoapp.localhost`
- GraphQL endpoint: `http://api.todoapp.localhost/graphql`
- GraphiQL: `http://api.todoapp.localhost/graphiql`

Use a custom proxy port when needed:

```bash
DEV_PROXY_PORT=1355 make start
```

Then access with `:1355` (for example `http://todoapp.localhost:1355`).

## Testing

API integration tests are run with **Hurl**:

```bash
make hurl
```

By default this targets `http://APP_SERVER__IP:APP_SERVER__PORT` (defaults to `http://127.0.0.1:3000`), matching the Make-exported app settings.

To test a different backend port:

```bash
APP_SERVER__PORT=4000 make hurl
```
